03 · Reproduce a grade

Check our work against DNS.

A grade is deterministic: the same public DNS records always produce the same grade, by the rubric in the Methodology. You can read the same records we do with any DNS-over-HTTPS client — no account, no tooling of ours required.

# the four lookups behind a grade for example.com curl -s 'https://dns.google/resolve?name=example.com&type=TXT' # SPF (v=spf1 … all) curl -s 'https://dns.google/resolve?name=_dmarc.example.com&type=TXT' # DMARC (p=…) curl -s 'https://dns.google/resolve?name=example.com&type=MX' # MX presence curl -s 'https://dns.google/resolve?name=google._domainkey.example.com&type=TXT' # DKIM (per selector)

Apply the point values from the scoring table and you will arrive at the same grade our in-browser domain check shows — there is no hidden state.

04 · Reproduce the record

Rebuild the Observatory dataset.

The Observatory record is produced by an open script over the canonical domain set, re-measured weekly, with an immutable archive so any past state can be recovered from version history.

# from a clone of the repository node scripts/observe.js # writes observatory/latest.json + history/<date>.json node scripts/verify.js # the same deterministic gate CI runs

If any domain fails to resolve during a scheduled run, the run aborts rather than publish a partial record — measurement integrity over completeness, as stated in the Methodology.
