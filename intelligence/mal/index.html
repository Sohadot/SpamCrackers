Email Abuse Intelligence · Pillar MAL

Malware Delivery.

Mail is the oldest and busiest delivery route for malicious code — from macro documents to HTML smuggling and password-gated archives. This pillar models how a payload is packaged, delivered, evaded, executed and entrenched. The fourth pillar of Email Abuse Intelligence.

Namespace EAI · MAL Release v1.0 Posture Defensive & analytical Techniques 29
01 · Pillar thesis

The envelope is the exploit.

For decades the fastest way onto a machine has been to ask a person to open something. Malware delivery is less about the code and more about the container and the coax — the macro prompt, the shortcut that looks like a document, the archive whose password sits helpfully in the email. The gateway that only inspects the obvious misses the rest.

This pillar treats delivery as a chain — packaging, delivery, evasion, execution, persistence and impact — so defenders can intervene before code runs, not only after. It bridges to the spam pillar for origin infrastructure and to the phishing pillar where the lure and the payload share a message.

02 · Scope & boundaries

Defensive by definition.

What this is
  • A classification and reference model for how malware is delivered by mail.
  • Analytical and defensive — for defenders, responders and researchers.
  • Focused on observable signals and controls.
  • Open and citable, with stable identifiers.
What this is not
  • Not malware, a builder, or any payload of any kind.
  • Not operational instructions for creating or delivering malware.
  • Not code, samples, or steps to reproduce an attack.
  • Not a substitute for lawful, authorised testing.

Every technique is described at the level of classification and defence — what it is, how it is observed, and how it is countered. No samples, code or build steps appear anywhere in this model.

03.1 · Malware-delivery taxonomy

Four axes of classification.

A delivery is described by its payload class, delivery mechanism, execution trigger and evasion posture.

By payload class

AA2Info-stealerHarvests credentials, cookies, wallets and files.
AA3RansomwareEncrypts or exfiltrates data for extortion.
AA4Remote-access trojanGrants an operator interactive control of the host.
AA5Banking trojanTargets financial credentials and sessions.
AA6Botnet / wormEnlists the host into a network for reuse or spread.

CodeClassDefinition

BB3Link to payloadA URL leading to a hosted download.

MAL-PK

Packaging & Weaponisation

How a payload is wrapped to survive the journey to the inbox.

MAL-PK-01

Direct executable attachment

What it is
An executable is attached, often disguised by icon or double extension.
Observable signals
Executable or double-extension files attached to mail.
Defensive response
Block executable attachment types at the gateway by policy.
MAL-PK-02

Macro-enabled document

What it is
An office document carries macros that fetch or run the payload when enabled.
Observable signals
Macro-enabled formats prompting the user to enable content.
Defensive response
Disable macros from the internet by policy; sandbox documents.
In patternsEAP-05
MAL-PK-03

Shortcut & script files

What it is
LNK shortcuts or script files (JS, HTA, VBS) invoke an interpreter to run code.
Observable signals
Shortcut or scripting file types delivered by mail.
Defensive response
Restrict script hosts and block risky file types.
MAL-PK-04

HTML smuggling package

What it is
A benign-looking HTML file assembles the payload in the browser to bypass gateways.
Observable signals
HTML attachments that build a download client-side.
Defensive response
Render attachments in a sandbox; restrict HTML attachments.
In patternsEAP-06
MAL-PK-05

Password-protected archive

What it is
The payload is placed in an archive whose password is given in the email body, keeping scanners out.
Observable signals
An archive with the password supplied in the message.
Defensive response
Inspect archive contents; treat mailed archive passwords as a red flag.
MAL-DL

Delivery

How the packaged payload actually reaches the recipient.

MAL-DL-01

Attachment delivery

What it is
The payload rides directly in the message as a file.
Observable signals
Risky file types arriving as attachments.
Defensive response
Attachment sandboxing and type policy at the gateway.
In patternsEAP-05
MAL-DL-02

Link to hosted payload

What it is
A link leads to a download rather than carrying the file.
Observable signals
Links to direct downloads, especially of risky types.
Defensive response
URL detonation and download inspection before delivery.
MAL-DL-03

Cloud-service staging

What it is
The file is hosted on a reputable cloud service to borrow its trust.
Observable signals
Payloads staged on file-sharing or cloud platforms.
Defensive response
Detonate cloud-hosted links; reputation is not safety.
MAL-DL-04

Reply-chain delivery

What it is
The lure is inserted into a genuine thread to raise trust (bridges PHI-LP-05).
Observable signals
An unexpected attachment or link resuming a real thread.
Defensive response
Scan links and files even within known conversations.
In patternsEAP-06
MAL-DL-05

Malvertising redirect

What it is
Delivery arrives via an advertising or redirect network to the payload.
Observable signals
Redirect chains terminating in a download.
Defensive response
Follow and detonate full redirect chains.
MAL-EV

Evasion & Anti-analysis

How the payload avoids automated inspection.

MAL-EV-01

Sandbox evasion

What it is
Execution is withheld when analysis environments are detected.
Observable signals
Content that behaves differently for an analyser than a user.
Defensive response
Realistic detonation environments and delayed re-analysis.
In patternsEAP-05
MAL-EV-02

Packing & obfuscation

What it is
Code is compressed or scrambled to defeat static signatures.
Observable signals
High-entropy or obfuscated files with little static detail.
Defensive response
Behavioural analysis and unpacking rather than signatures alone.
MAL-EV-03

Password gating

What it is
An archive password in the email keeps gateway scanners out of the payload.
Observable signals
Mailed passwords accompanying archives.
Defensive response
Extract with the supplied password and inspect; flag the pattern.
In patternsEAP-05
MAL-EV-04

Geo & time gating

What it is
The payload serves only to certain regions or times to dodge analysis.
Observable signals
Content varying by geography or hour of request.
Defensive response
Analyse from multiple vantage points and over time.
MAL-EV-05

Benign-first staging

What it is
Early stages appear harmless; capability is fetched only later.
Observable signals
Innocuous droppers with delayed network activity.
Defensive response
Egress monitoring and staged-behaviour detection.
MAL-EX

Execution & Initial Access

How code first runs on the recipient's machine.

MAL-EX-01

Macro execution

What it is
Enabled macros run the fetch-and-execute logic.
Observable signals
Documents spawning script interpreters or network calls.
Defensive response
Block internet-macro execution; monitor office-to-child processes.
In patternsEAP-05
MAL-EX-02

Script-host execution

What it is
A shortcut or script triggers an interpreter to run the payload.
Observable signals
Script hosts launched from mail-delivered files.
Defensive response
Constrain or disable script hosts; application control.
MAL-EX-03

Document exploit

What it is
A viewer vulnerability runs code without user action.
Observable signals
Crafted documents targeting known viewer flaws.
Defensive response
Timely patching and exploit-mitigation controls.
MAL-EX-04

User-run persuasion

What it is
Social engineering convinces the user to run the file themselves.
Observable signals
Instructions coaching the user to open or enable.
Defensive response
User awareness plus application allow-listing.
In patternsEAP-06
MAL-EX-05

Living-off-the-land

What it is
Trusted system binaries are abused to execute or fetch the payload.
Observable signals
Unusual use of legitimate system tools.
Defensive response
Behavioural detection of anomalous native-tool use.
MAL-IN

Install & Persistence

How the payload establishes itself once it runs.

MAL-IN-01

Dropper / loader stage

What it is
A first stage installs and launches the main payload.
Observable signals
Small executables fetching larger second stages.
Defensive response
Endpoint detection and egress controls on new processes.
In patternsEAP-05
MAL-IN-02

Autostart persistence

What it is
Registry keys or scheduled tasks re-launch the payload after reboot.
Observable signals
New autostart entries or scheduled tasks.
Defensive response
Monitor persistence locations; baseline autostart state.
In patternsEAP-05
MAL-IN-03

DLL side-loading

What it is
A trusted application is made to load a malicious library.
Observable signals
Legitimate binaries loading unexpected libraries.
Defensive response
Application control and library-load monitoring.
MAL-IN-04

Defence tampering

What it is
Security tooling is disabled or blinded to protect the foothold.
Observable signals
Attempts to stop or exclude security agents.
Defensive response
Tamper protection and independent, off-host alerting.
MAL-OB

Objective & Impact

What the malware is ultimately there to do.

MAL-OB-01

Data theft

What it is
Credentials, cookies, files and secrets are exfiltrated.
Observable signals
Bulk data access and unusual outbound transfer.
Defensive response
Data-loss prevention and egress anomaly detection.
MAL-OB-02

Ransomware impact

What it is
Data is encrypted or stolen for extortion.
Observable signals
Mass file changes and shadow-copy deletion.
Defensive response
Tested backups, segmentation and rapid isolation.
In patternsEAP-05
MAL-OB-03

Remote control

What it is
An operator gains interactive command of the host.
Observable signals
Command-and-control beacons and remote sessions.
Defensive response
Network detection and command-and-control blocking.
In patternsEAP-06
MAL-OB-04

Lateral movement

What it is
The foothold is used to reach further systems.
Observable signals
Internal reconnaissance and credential reuse.
Defensive response
Segmentation, least privilege and identity monitoring.
MAL-OB-05

Onward propagation

What it is
The host is used to send further malicious mail (bridges SPM-IA-02).
Observable signals
Outbound malware or spam from the compromised host.
Defensive response
Outbound filtering and rapid containment.
03.3 · Indicators & evidence model

No claim without an observable.

The malware pillar shares the Email Abuse Intelligence evidence model — with file and behavioural signals carrying particular weight.

CodeClassEvidence captured
EAI-EV-NETNetworkSending IP, ASN, reverse DNS, download and command-and-control endpoints.
EAI-EV-AUTHAuthenticationSPF, DKIM and DMARC results for the sending identity.
EAI-EV-CONTContentAttachment and file hashes, file types, macro and script indicators, URL patterns.
EAI-EV-BEHAVBehaviouralProcess lineage, script-host activity, persistence changes, egress patterns.
EAI-EV-INFRAInfrastructurePayload-hosting domains and age, redirect chains, certificate and registrar detail.
Confidence model
LowA single weak or ambiguous indicator. Suitable for enrichment, not enforcement.
ModerateMultiple independent indicators from different classes agree.
HighCorroborated indicators plus file or behavioural evidence.
ConfirmedDirect, reproducible evidence — e.g. a detonated, observed payload.
