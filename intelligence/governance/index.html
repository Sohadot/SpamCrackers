Email Abuse Intelligence · Governance

Model Governance.

A reference is only as trustworthy as its rules. This is how the Email Abuse Intelligence model is identified, versioned, changed and cited — deterministic, transparent and defensive by design, so anyone can depend on it and cite it without ambiguity.

01 · Principles

Four rules the model never breaks.

Deterministic

Every element has one stable identifier. The same input always maps to the same reference — no ambiguity, no drift.

Transparent

Grading, taxonomy and technique criteria are published in full. There is no private model behind the public one.

Stable

Identifiers are permanent. Once published, an id is never reused or renumbered; retired entries are deprecated, not deleted.

Defensive

Every entry is classification-and-defence only. Operational content is out of scope by policy, at every version, without exception.

Every element has one permanent name.

Identifiers are structured, human-readable and stable. This is the full grammar of the EAI namespace.

Identifier-stability policy

Published identifiers are permanent. An id is never reused for a different concept and never renumbered. When a technique is superseded it is marked deprecated and kept in place, so existing citations never break. New entries take the next free number within their phase.

03 · Versioning

Semantic versioning for a living model.

The model is versioned MAJOR.MINOR.PATCH. The current release is v1.0.0.

MAJOR

A breaking change to the model's structure or identifier scheme. Rare, and never silent.

MINOR

Additive change — a new pillar, phase, technique or taxonomy class. Existing ids are untouched.

PATCH

Clarifications, wording and defensive-guidance updates. No ids change.
