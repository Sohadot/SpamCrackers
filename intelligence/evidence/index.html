Reference Model · Evidence & Confidence v1.0

No claim without
an observable.

A technique or pattern is a hypothesis until evidence supports it. This model is the reference's substantiation layer: five classes of observable evidence and four tiers of confidence — each with a stable identifier and a precise rule, so an assessment can be stated, cited, and challenged on the same terms.

01 · Evidence classes

Five kinds of observable.

Every indicator that can substantiate a classification falls into one of five classes. Each carries a stable EAI-EV-* identifier and is noted with where it can actually be observed — because the class an analyst reasons about is not always one a given tool can see.

IdentifierClassEvidence captured
EAI-EV-NET Network Sending IP, ASN, reverse DNS, netblock and geolocation, HELO/EHLO identity.
Observed From the receiving MTA and message headers — not from DNS alone.
EAI-EV-AUTH Authentication SPF result, DKIM result and selector, DMARC policy and alignment, ARC chain.
Observed The one class SpamCrackers reads directly: SPF, DMARC and MX exactly, and DKIM heuristically, over public DNS. A pass/fail result and ARC require the received message.
EAI-EV-CONT Content URL and domain patterns, file and body hashes, template and layout fingerprints, header anomalies.
Observed From the message body and its attachments.
EAI-EV-BEHAV Behavioural Volume and velocity, send timing, list-bombing bursts, engagement tracking.
Observed From aggregate mail-flow telemetry over time.
EAI-EV-INFRA Infrastructure Domain age and registrar, nameservers, hosting, redirect chains, certificate history.
Observed From DNS, WHOIS and certificate-transparency enrichment.
02 · Confidence tiers

How evidence becomes confidence.

Indicators combine into a graded assessment. The tiers are ordered and their rules are explicit — a higher tier requires strictly more, and stronger, corroboration than the one below it.

Low EAI-CONF-1 level 1 / 4
Rule

A single weak or ambiguous indicator from one evidence class.

Use

Suitable for enrichment and triage — not for enforcement.

Moderate EAI-CONF-2 level 2 / 4
Rule

Multiple independent indicators from two or more different evidence classes point to the same classification.

Use

Supports prioritisation and soft action such as flagging.

High EAI-CONF-3 level 3 / 4
Rule

Corroborated indicators across classes, including at least one strong class — Authentication or Infrastructure.

Use

Supports enforcement decisions.

Confirmed EAI-CONF-4 level 4 / 4
Rule

Direct, reproducible evidence — a detonated payload, an observed campaign artefact, or a reproduced authentication failure.

Use

Definitive; supports attribution-grade conclusions and blocking.

03 · The rule

One principle, applied consistently.

No classification is asserted without an observable. A single indicator earns the lowest tier; confidence rises only as independent indicators from different classes corroborate one another, and reaches its ceiling only with direct, reproducible evidence. Corroboration across classes matters more than volume within one — ten content signals of the same kind are weaker than one content signal aligned with an authentication and an infrastructure signal.

This is why the reference model describes every technique in terms of its observable signals: the model and this evidence layer are the same discipline seen from two sides — what to look for, and how much to trust what you find.

04 · Honest scope

What this site actually observes.

The evidence model is the full analytical framework a defender reasons within. SpamCrackers' own automated tooling sees only part of it — and says so.

SpamCrackers directly observes the Authentication class only (EAI-EV-AUTH), and only its DNS-visible part: SPF, DMARC and MX exactly, DKIM heuristically — as set out in the Methodology. The Network, Content, Behavioural and Infrastructure classes require the received message or telemetry a static, in-browser tool does not have. They are published here as the reference framework for defenders who do hold that evidence — not as something this site measures.

Stating the boundary is the point: a grade from the domain check is an Authentication-class observation at a single moment, and it should be read as exactly that — no more, no less.
